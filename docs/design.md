# 总体设计文档

## 一、项目概述

灵山胜境 AI 数字人导览系统面向第十五届“中国软件杯” A5 赛题，构建一个具备多模态交互能力的景区导览 AI 数字人：游客端提供数字人形象、语音/文本问答、个性化路线与情感互动；管理端提供知识库管理、数字人配置、运营数据分析与游客反馈洞察。景区数据以无锡灵山胜境官方公开资料为准（拈花湾作为可选子景点）。

## 二、总体架构

```
        游客端 (Vue3)                     管理端 (Vue3)
   数字人 / 问答 / 语音 / 路线 / 反馈     知识库 / 数字人 / 大屏 / 会话反馈 / 用户
             │  HTTP /api                     │
             ▼                                ▼
        ┌──────────────────────────────────────────┐
        │        Node.js API 服务 (统一信封)          │
        │  鉴权 · RAG 检索 · 大模型 · 语音 · 路线 · 分析 │
        └──────────────────────────────────────────┘
             │                     │                │
        SQLite 持久化        本地知识库(JSON)     云端能力(可选)
     会话/反馈/配置/用户/行为   景点/分块/向量     LLM · ASR · TTS
                                                讯飞交互式数字人(浏览器 SDK)
```

- **前端**：基于 art-design-pro（Vue3 + TypeScript + Element Plus + Vite）。游客端与管理端共用一套布局与路由，前端权限模式按角色渲染菜单。
- **数字人**：讯飞交互式虚拟人 SDK，在浏览器内建立 WebRTC 流，通过 `writeText(答案文本)` 由云端完成语音合成、口型同步与表情驱动；账号凭证来自环境变量，形象/音色由后台配置下发。
- **后端**：Node.js 原生 HTTP 服务，统一响应信封 `{ success, data, meta } / { success, error }`，Bearer Token 鉴权。核心能力为本地 RAG 问答、路线规划、语音链路封装与运营分析。
- **持久化**：SQLite（better-sqlite3）。会话、消息、反馈、景点、FAQ、数字人配置、行为分析、路线选择、消息标注、后台用户均入库；测试脚本使用内存库保证确定性。

## 三、核心功能设计

### 3.1 多模态交互
- 文本问答：`POST /api/chat/ask`，返回答案、质量标签、情绪标签、引用来源与各阶段耗时。
- 语音问答：浏览器 Web Speech 识别 → 问答 → 数字人播报；后端 `POST /api/voice/ask` 提供 ASR→RAG→TTS 链路封装。
- 数字人：根据答案情绪标签映射播报情感系数与表情，形象/音色可后台配置，异常时降级为文本模式。

### 3.2 智能问答与知识库（RAG）
- 依据官方灵山胜境资料构建结构化景点实体与知识分块，混合检索（关键词 + 内容覆盖 + 景点实体加权）并重排（开放时间/票价/路线/亮点意图优先）。
- 系统提示词限定“仅基于灵山胜境资料作答”，越界问题引导回景区，无资料问题诚实拒答并给出可追问方向。
- 后台固定问答（FAQ）优先于 RAG，命中缓存降低时延。

### 3.3 个性化路线
- 本地路线图与景点标签（历史/佛教/亲子/拍照/演艺等），按游玩时长与兴趣评分推荐经典/深度/亲子/拍照等路线，输出节点顺序、耗时与推荐理由，并支持点击节点由数字人讲解。

### 3.4 管理后台
- 知识库管理：文档上传与重建、景点与别名维护、FAQ 维护、检索与问答测试。
- 数字人配置：形象、音色、欢迎语、语速、情绪风格、服务状态，启用项实时下发游客端。
- 运营数据大屏：服务人次、满意度、平均时延、热门问答、景点热度、情感分布、消费结构、人群画像、运营建议，支持性别/年龄/满意度筛选。消费类数据来源于官方行为分析 Excel。
- 会话与反馈：会话明细、人工标注、错误问答生成知识补充草稿、反馈聚类、服务质量报告。
- 系统管理：管理员/运营账号的用户管理（产品闭环）。

## 四、关键指标与保障

- **准确性**：标准问答测试集 50 题自测脚本 `eval:chat`，检索 Top3 命中评测 `eval:retrieval`。
- **响应时延**：问答返回各阶段耗时，游客端展示；FAQ 缓存与首句快速返回优化“语音问答 < 5 秒”。
- **稳定性**：请求日志、超时重试、`DEMO_MODE` 演示降级、数字人加载失败降级为文本模式、前端异常边界。
- **数据安全**：管理端接口 Bearer 鉴权，密码使用 scrypt 加盐哈希。

## 五、技术选型理由

- 复用已验证的 Node.js 后端与本地知识库，保证问答准确性与演示稳定；SQLite 零配置、便于部署与备份。
- 讯飞交互式虚拟人贴合赛题“逼真、口型同步、表情配合”的要求，且形象/音色可按景区文化配置。
- art-design-pro 提供成熟的布局、权限与组件体系，便于快速构建游客端与管理端。

## 六、创新点

- 答案“情绪标签 → 数字人情感/表情”联动，实现情感互动。
- 检索结果引用来源可追溯，回答可解释。
- 运营数据大屏融合“线上问答行为 + 线下消费行为”双数据源，支撑营销决策。
- 全链路演示降级设计，保证比赛现场稳定。

## 七、后续可扩展方向

- 引入向量检索与更强云端大模型进一步提升准确率与自然度。
- 结合 GPS / 定位实现“到达即讲解”的伴随式导览。
- 语音/表情多模态情感识别，增强情感互动深度。
